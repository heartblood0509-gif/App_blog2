# Blog Pick 설치 매뉴얼 — Mac

> 이 문서는 **Mac 컴퓨터**에서 Blog Pick을 처음 설치하시는 분을 위한 안내입니다.
> 위에서부터 한 줄씩 따라하시면 30분 안에 첫 글 발행까지 가능합니다.

---

## ⚠️ 주의사항

이 도구는 **네이버의 자동 로그인·자동 발행** 기능을 사용합니다.

- 네이버 서비스 이용약관과 충돌할 가능성이 있으며, 이로 인한 **계정 제한·차단의 책임은 사용자에게 있습니다**.
- 캡차(보안 문자)·인증 우회 시도는 금지되어 있으므로, **캡차가 뜨면 직접 풀어주세요**.
- 사용 중 네이버에서 이상 접속 경고가 뜨면 **즉시 사용을 중단**하고 휴식 시간을 가지세요.
- **자동 발행은 하루 최대 4편을 권장합니다** — 계정 2개 × 계정당 2편이 안전선입니다. 글이 더 필요하시다면 블로그픽으로 본문만 생성한 뒤 **직접 네이버 블로그에 복사·붙여넣어** 발행해주세요. 그 이상 자동 발행을 무리하게 돌리면 계정 차단 위험이 커서 권장하지 않습니다.

이 안내에 동의하신 분만 다음 단계로 진행해주세요.

---

## 🧾 준비물 체크리스트

설치 시작 전 아래가 모두 준비됐는지 확인하세요.

- [ ] **macOS 11 (Big Sur) 이상** (최신 macOS 권장)
- [ ] **네이버 계정** (글을 올릴 네이버 블로그 계정)
- [ ] **Gemini API 키** (무료 발급, 아래 1단계에서 안내)
- [ ] **인터넷 연결** (안정적인 연결 필요)
- [ ] (선택) **공식 배포처에서 받은 설치 파일** (.dmg)

---

## 1단계. Gemini API 키 발급 (5분)

블로그픽이 AI로 글을 쓰려면 Google의 Gemini API 키가 필요합니다.

1. 웹브라우저(Safari·Chrome 무관)에서 **`https://aistudio.google.com/app/apikey`** 접속
2. **Google 계정으로 로그인** (기존 Gmail 계정 사용 가능)
3. **"Create API key" (API 키 만들기)** 버튼 클릭
4. (창이 비어 있고 키 생성 버튼이 안 보이는 경우) **"Projects" → "Import projects"** 로 들어가 기존 프로젝트를 가져온 뒤 다시 시도
5. 발급된 키(`AIzaSy...`로 시작하는 긴 문자열)를 **메모장이나 안전한 곳에 임시 저장**
   - ⚠️ 키는 비밀번호처럼 다뤄야 합니다. 절대 SNS·공개 채팅에 올리지 마세요.

**💰 비용 안내**
- 무료 한도 내에서 사용 가능합니다.
- 사용량이 많아지면 결제 설정이 필요할 수 있습니다.
- 현재 요금 정책: https://ai.google.dev/gemini-api/docs/billing

---

## 2단계. 받을 파일 확인하기

블로그픽 맥 버전은 **Apple Silicon(M1~M4 칩)** 용 하나예요. 요즘 맥은 대부분 여기에 해당합니다.

- 받을 파일: **`Blog-Pick-Mac`** 이 들어간 `.dmg` 파일 (`Blog-Pick-Mac-XXX.dmg`)

---

## 3단계. 설치 파일 다운로드

공식 배포처에서 **`Blog-Pick-Mac-….dmg`** 파일을 받으세요.

- **`Blog-Pick-Mac-XXX.dmg`**

> 📥 **다운로드 링크:** `[배포 담당자가 채울 자리]`

다운로드한 파일의 **크기·해시값**이 배포처에 적힌 값과 같은지 한번 확인하면 안전합니다.

---

## 4단계. .dmg 파일 열고 응용 프로그램으로 드래그

1. 다운로드한 `.dmg` 파일을 **더블클릭**
2. 작은 창이 열리며 다음이 보입니다:
   - 좌측: **Blog Pick 아이콘**
   - 우측: **응용 프로그램(Applications) 폴더**
3. Blog Pick 아이콘을 **응용 프로그램 폴더로 드래그** (이게 "설치" 입니다)
4. 복사가 끝나면 .dmg 창을 닫고, 바탕화면의 .dmg 아이콘을 **꺼내기(eject)** 또는 휴지통으로 이동

---

## 5단계. 첫 실행 시 "확인되지 않은 개발자" 경고 ⚠️ 여기서 가장 많이 막힙니다

응용 프로그램 폴더에서 Blog Pick을 **더블클릭**하면 처음에는 아래 같은 경고가 뜹니다.

> ❗ **"확인되지 않은 개발자가 만든 앱이라 열 수 없습니다"**

**👉 공식 배포처에서 받은 파일이 맞다면, 아래 두 방법 중 하나로 여세요:**

### 방법 A — 간단 (권장)
1. **Finder** 열고 응용 프로그램 폴더로 이동
2. Blog Pick 아이콘 위에서 **`Control` 키를 누른 채로 클릭** (또는 우클릭/두 손가락 클릭)
3. 메뉴에서 **"열기"** 선택
4. 다시 한번 뜨는 경고 창에서 **"열기"** 클릭

### 방법 B — 시스템 설정에서 허용
1. **Apple 메뉴 → "시스템 설정"** 열기
2. **"개인정보 보호 및 보안"** 클릭
3. 아래로 스크롤하면 **`"Blog Pick"이(가) 차단되었습니다`** 라는 문구가 보입니다
4. 옆의 **"그래도 열기"** 버튼 클릭

> 💡 이 작업은 **처음 한 번만** 하면 됩니다. 두 번째부터는 일반 앱처럼 열립니다.

> 🛡 이 경고는 Apple 공증을 받지 않은 모든 앱에 똑같이 나오는 일반적인 안내입니다.
> 단, **출처가 불분명한 파일이라면 절대 열지 마세요**.

---

## 6단계. 권한 요청 허용

처음 실행하면 몇 가지 권한 요청 알림이 뜹니다. 모두 **"허용"** 또는 **"항상 허용"** 을 누르세요.

- **데스크탑 폴더 접근** → "허용"
- **키체인 접근** → "**항상 허용**" (네이버 비밀번호를 안전하게 저장하는 데 필요)
- (일부 macOS 버전) **접근성(Accessibility) 권한** 요청 시:
  → 시스템 설정 → 개인정보 보호 및 보안 → **접근성** → Blog Pick **체크**

---

## 7단계. 로그인 + API 키 등록

Blog Pick이 열리면 로그인 화면이 보입니다.

1. 가입 시 사용한 **이메일/비밀번호**로 로그인
2. 로그인 후 화면 우측 상단의 **🔑 열쇠 아이콘** 클릭 → "API 키 설정" 페이지
3. 1단계에서 저장해둔 **Gemini API 키**를 입력칸에 붙여넣기
4. **"저장"** 클릭

---

## 8단계. 네이버 블로그 계정 등록

발행할 네이버 계정을 등록합니다.

1. 네이버 ID/비밀번호 입력
2. 비밀번호는 Mac의 **키체인에 암호화되어 저장**됩니다 (외부 전송 없음)
3. ⚠️ **네이버 2단계 인증**을 켜둔 계정이면 발행 시 **OTP 입력 창**이 뜰 수 있습니다 → 휴대폰을 곁에 두세요
4. 보안을 위해 **2단계 인증은 계속 켜둘 것을 권장**합니다 (해제하지 마세요)

---

## 9단계. 첫 글 발행해보기

이제 준비 완료. 앱 상단의 **❓ 도움말 버튼**을 클릭하면 사용 매뉴얼이 나옵니다.
거기 안내대로 한 편 발행해보세요.

---

## 🔄 자동 업데이트 안내 (Mac은 수동 업데이트)

> ⚠️ **Mac은 현재 자동 업데이트가 비활성화**되어 있습니다.

새 버전이 나오면:
1. 배포처에서 새 `.dmg` 파일을 다시 받기
2. 위 **4단계 ~ 5단계** 와 동일한 방법으로 다시 설치 (응용 프로그램으로 드래그 → 덮어쓰기)
3. **등록한 계정·이미지·보관함은 그대로 유지됩니다** (앱 본체만 교체되는 것이라)

> 📝 차후 Apple Developer 인증을 도입하면 Windows처럼 자동 업데이트로 전환될 예정입니다.

## ❓ 자주 묻는 질문 (FAQ)

**Q. "손상된 파일이라 휴지통으로 이동" 경고가 떠요**
A. 경고창은 **"취소"** 를 누르고(휴지통으로 보내지 마세요), 위 **5단계의 "방법 B — 시스템 설정 → 개인정보 보호 및 보안 → 그래도 열기"** 를 따라 하세요. 대부분 해결됩니다. (아주 오래된 macOS에서 그래도 안 열리면, 인앱 도움말의 "터미널" 최후수단을 참고하세요.)

**Q. "이 앱은 macOS XX에서 작동하지 않을 수 있습니다" 메시지가 떠요**
A. macOS 11 미만이면 지원되지 않습니다. macOS를 업데이트하시거나 다른 컴퓨터에서 사용하세요.

**Q. Gemini API 무료 한도가 초과됐어요**
A. 다음 날 자동으로 풀리거나, Google Cloud에서 결제 연결로 한도를 늘릴 수 있습니다.

**Q. 발행 중 캡차(보안 문자)가 떴어요**
A. 자동화 창에 캡차가 보이면 **직접 풀어주세요**. 자동 우회는 약관 위반입니다.

**Q. 회사 컴퓨터에서 쓰려고 합니다. 방화벽이 막아요**
A. 회사 IT팀에게 다음 도메인을 허용해달라고 요청하세요:
- `*.googleapis.com` (Gemini API)
- `*.naver.com`, `*.pstatic.net` (네이버 발행)
- 인증 도메인 (앱 시작 시 로그인용)

## 🗑 완전 삭제 (데이터까지 모두 지우고 싶을 때)

> ⚠️ **경고**: 아래 절차를 진행하면 **등록한 네이버 계정, API 키, 생성한 이미지, 보관함**이 전부 사라집니다.
> 백업이 필요하다면 사용 매뉴얼의 "백업·PC 이전" 섹션을 먼저 보세요.

1. 응용 프로그램 폴더에서 **Blog Pick**을 **휴지통**으로 이동
2. Finder 메뉴바에서 **"이동" → "폴더로 이동"** (단축키: `Cmd + Shift + G`)
3. 입력칸에 **`~/Library/Application Support/app-blog2-desktop`** 입력 → 엔터
4. 열린 폴더를 **통째로 삭제**
5. (선택) 같은 방법으로 **`~/Library/Logs/app-blog2-desktop`** 도 확인 후 삭제
6. 휴지통 비우기
