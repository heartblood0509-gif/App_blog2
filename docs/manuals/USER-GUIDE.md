# Blog Pick 사용 매뉴얼

> 앱을 처음 켜셨다면, 이 매뉴얼을 위에서 아래로 차근차근 읽어주세요.
> **5단계만 따라 하시면 글 한 편이 네이버 블로그에 자동으로 올라갑니다.**

---

## ⚠️ 주의사항

이 도구는 **네이버의 자동 로그인·자동 발행** 기능을 사용합니다.

- 네이버 서비스 이용약관과 충돌할 가능성이 있으며, 이로 인한 **계정 제한·차단의 책임은 사용자에게 있습니다**.
- 캡차(보안 문자)·인증 우회 시도는 금지되어 있으므로, **캡차가 뜨면 직접 풀어주세요**.
- 사용 중 네이버에서 이상 접속 경고가 뜨면 **즉시 사용을 중단**하고 휴식 시간을 가지세요.
- **자동 발행은 하루 최대 4편(계정 2개 × 계정당 2편)을 권장합니다.** 글이 더 필요하시다면 블로그픽으로 본문만 생성한 뒤 **직접 네이버 블로그에 복사·붙여넣어** 발행해주세요. 그 이상 자동 발행을 무리하게 돌리면 계정 차단 위험이 커서 권장하지 않습니다.

셋 중 하나라도 안 됐으면 **먼저 설치 매뉴얼**을 다시 보세요.

## ✍️ 글쓰기 5단계 위저드

### 1단계. 글쓰기 모드 고르기

| 모드 | 어떤 글에 좋을까 |
|---|---|
| **SEO/AEO 통합형** | 네이버 검색 + AI 검색(ChatGPT·Gemini)에 잘 잡히는 글. **대부분 이걸 고르시면 됩니다.** |
| **브랜드 블로그** | 우리 브랜드 톤·스타일을 일관되게 유지하면서 쓰는 글. **브랜드 프로필**을 미리 만들어두면 자동 반영됩니다. |

> 모드에 따라 다음 단계가 살짝 달라집니다.

### 2단계. 주제·키워드·제품 입력

- **키워드형**: "겨울철 두피 관리" 같은 메인 키워드 한 줄
- **제품형**: 미리 등록해둔 제품 중 선택 → 제품 정보가 글에 자동 반영
- **제목 공식** (선택): 예) "[질문형 제목]" 을 입력하면 AI가 그 공식대로 제목을 만듭니다

### 3단계. 생성 (AI가 글을 씁니다)

- 보통 **30초 ~ 2분** 소요. 닫지 말고 기다리세요.
- 진행률이 보이면 정상 작동 중입니다.
- 에러가 나면: 보통 **API 키 한도 초과** 또는 **인터넷 끊김**. API 키 확인 후 재시도.

### 4단계. 검토·편집

생성된 글이 **SmartEditor** (네이버 블로그 에디터와 동일한 UI)에 나옵니다.

자유롭게 수정하셔도 됩니다. 단, **다음만 절대 누르지 마세요** ⚠️

> 🚨 **Cmd+A (Mac) / Ctrl+A (Win) → Delete**
> 전체 선택 후 한 번에 지우면 편집기가 **흰 화면**이 되는 알려진 문제가 있습니다.
> 글을 통째로 지우고 싶다면 → 모달을 닫고 처음부터 다시 생성하는 게 안전합니다.

- **이미지 첨부**: 자동으로 들어간 이미지 외에 직접 추가 가능
- **제품 첨부**: 제품 카드를 글 중간중간 끼워 넣을 수 있음

### 5단계. 발행

1. 발행할 **네이버 계정 선택**
2. **카테고리·태그·공개 설정** 확인
3. **"발행"** 버튼 클릭
4. 자동화 창이 잠깐 떴다가 닫힙니다 → **그 동안 키보드·마우스 건드리지 마세요**
5. 발행 완료되면 토스트 메시지와 함께 **글 URL** 이 표시됩니다

## 🚨 자주 막히는 함정 (꼭 한번 읽어주세요)

**네이버 2단계 인증을 켜뒀어요**
→ 발행할 때 OTP 입력 창이 떠요. 휴대폰 보안 앱을 켜두세요.
→ **2단계 인증은 끄지 않는 것을 권장**합니다 (계정 보호 우선).
→ 자동 발행 빈도가 잦으면 발행 전용 부계정을 따로 만드시되, 그 부계정도 2단계 인증은 유지하세요.

**네이버가 "비정상 접속"으로 차단했어요**
→ 같은 IP에서 너무 자주 자동 발행하면 일시적으로 막힙니다.
→ **하루 10편 이하** 권장. 차단되면 한동안 사용을 멈추고 자연스럽게 회복되길 기다리세요.

**SmartEditor가 흰 화면이 됐어요**
→ 위의 ⚠️ 경고(Ctrl+A → Delete)를 안 지키신 경우입니다.
→ 모달을 닫고 글을 다시 생성하세요. 저장된 이미지는 보관함에 남아 있을 수 있습니다.

**발행이 중간에 멈췄어요**
→ 캡차가 떴을 가능성이 큽니다. 자동화 창에 캡차가 보이면 **직접 풀어주시면** 이어집니다.
→ 자동화 창을 닫지 마세요.

**API 키 한도 초과 에러가 나요**
→ Gemini는 무료 한도가 있습니다.
→ 다음 날 자동으로 풀리거나, Google Cloud에서 결제 연결로 한도를 늘릴 수 있습니다. 자세한 요금: https://ai.google.dev/gemini-api/docs/billing

## 💾 내 데이터 백업·다른 PC로 옮기기

블로그픽의 모든 데이터(등록한 제품, 브랜드 프로필, 보관함 이미지 등)는 다음 폴더에 있습니다.

| OS | 데이터 위치 |
|---|---|
| **Windows** | `%APPDATA%\app-blog2-desktop\` (파일 탐색기 주소창에 그대로 입력) |
| **Mac** | `~/Library/Application Support/app-blog2-desktop/` (Finder → 이동 → 폴더로 이동) |

### 백업하기

1. 위 폴더를 통째로 **압축** (zip)
2. 클라우드·외장하드 등 안전한 곳에 보관

### 다른 PC로 옮기기 (PC 교체 시)

1. 기존 PC에서 위 폴더를 압축 → 새 PC로 옮기기
2. 새 PC에 Blog Pick **설치 → 종료** (한 번 실행해서 폴더 생성 후 종료)
3. 새 PC의 동일 경로 폴더에 백업본을 풀어 덮어쓰기
4. 새 PC에서 Blog Pick 실행

> ⚠️ **주의 사항**
> - **네이버 비밀번호는 OS 키체인 기반**이라 새 PC에서 **반드시 재등록**해야 합니다. (보안상 의도된 동작)
> - **Gemini API 키**는 파일로 옮겨가므로 재등록 불필요.
> - 새 PC에서 로그인할 때 **"기기 관리"** 에서 기존 PC 등록을 해제하거나, 다른 기기로 추가하세요.

## 🏢 회사 컴퓨터/회사 망에서 쓰시나요?

블로그픽은 인터넷으로 다음 도메인에 접속합니다. 회사 방화벽에 막힌다면 **IT팀에 아래 화이트리스트를 요청**하세요.

| 도메인 | 용도 |
|---|---|
| `*.googleapis.com` | Gemini AI 호출 |
| `*.naver.com`, `*.pstatic.net` | 네이버 로그인·발행 |
| `github.com`, `objects.githubusercontent.com` | 자동 업데이트 (Windows만) |
| 인증 서비스 도메인 | 앱 로그인 |

**프록시 환경**
- 시스템 프록시 설정을 따릅니다.
- PAC 스크립트·인증 프록시 환경은 정상 작동을 보장하지 않습니다.

**오프라인에서는?**
- 글 생성·발행 모두 **불가능** (AI와 네이버에 연결되어야 합니다)
- 다만 **보관함의 기존 이미지·글 열람**은 가능합니다

## 🆘 문제가 생기면

### 1) 로그 파일 위치

문의 전, 아래 로그 파일을 첨부해주시면 훨씬 빠르게 해결됩니다.

| OS | 로그 위치 |
|---|---|
| **Windows** | `%APPDATA%\app-blog2-desktop\logs\` 안의 **`main.log`**, **`backend.log`** |
| **Mac** | `~/Library/Logs/app-blog2-desktop/` 또는 `~/Library/Application Support/app-blog2-desktop/logs/` |

### 2) 🚨 로그·캡처를 공유할 때 절대 보내지 말 것

- **네이버 비밀번호** 원문
- **Gemini API 키 전체 값** (앞 4자리·뒤 4자리만 보이게 가리고 보내주세요. 예: `AIza...A1B2`)
- 개인 정보가 담긴 생성 이미지
