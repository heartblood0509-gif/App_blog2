## 📄 블로그-픽: 설치 매뉴얼 [윈도우]
# Blog Pick 설치 매뉴얼 — 📘Windows ver
> 이 문서는 Windows 컴퓨터에서 Blog Pick을 처음 설치하시는 분을 위한 안내입니다. 위에서부터 한 줄씩 따라하시면 30분 안에 첫 글 발행까지 가능합니다.
---
## ‣ 주의사항
이 도구는 네이버의 자동 로그인·자동 발행 기능을 사용합니다.
- 네이버 서비스 이용약관과 충돌할 가능성이 있으며, 이로 인한 계정 제한·차단의 책임은 사용자에게 있습니다.
- 캡차(보안 문자)·인증 우회 시도는 금지되어 있으므로, 캡차가 뜨면 직접 풀어주세요.
- 사용 중 네이버에서 이상 접속 경고가 뜨면 즉시 사용을 중단하고 휴식 시간을 가지세요.
- 자동 발행은 하루 최대 4편을 권장합니다 — 계정 2개 × 계정당 2편이 안전선입니다. 글이 더 필요하시다면 블로그픽으로 본문만 생성한 뒤 직접 네이버 블로그에 복사·붙여넣어 발행해주세요. 그 이상 자동 발행을 무리하게 돌리면 계정 차단 위험이 커서 권장하지 않습니다.
이 안내에 동의하신 분만 다음 단계로 진행해주세요.
---
## 🧾 준비물 체크리스트
설치 시작 전 아래가 모두 준비됐는지 확인하세요.
- [ ] Windows 10 이상 (Windows 11 권장)
- [ ] 네이버 계정 (글을 올릴 네이버 블로그 계정)
- [ ] Gemini API 키 (무료 발급, 아래 1단계에서 안내)
- [ ] 인터넷 연결 (안정적인 연결 필요)
- [ ] (선택) 공식 배포처에서 받은 설치 파일 (.exe)
---
## 1단계. Gemini API 키 발급 (5분)
블로그픽이 AI로 글을 쓰려면 Google의 Gemini API 키가 필요합니다.
1. 웹브라우저에서 https://aistudio.google.com/app/api-keys 접속
1. Google 계정으로 로그인 (기존 Gmail 계정 사용 가능)
1. "Create API key" (API 키 만들기) 버튼 클릭
1. (창이 비어 있고 키 생성 버튼이 안 보이는 경우) "Projects" → "Import projects" 로 들어가 기존 프로젝트를 가져온 뒤 다시 시도
1. 발급된 키(AIzaSy...로 시작하는 긴 문자열)를 메모장이나 안전한 곳에 임시 저장
- ⚠️ 키는 비밀번호처럼 다뤄야 합니다. 절대 SNS·공개 채팅에 올리지 마세요.
💰 비용 안내
- 무료 한도 내에서 사용 가능합니다.
- 사용량이 많아지면 결제 설정이 필요할 수 있습니다.
- 현재 요금 정책: https://ai.google.dev/gemini-api/docs/billing
- 유료 API 발급 방법 및 더 상세한 설명보기 ← 클릭

Tip

- 다운로드 실패 시, 주소를 복사하여 크롬 브라우저 최상단 주소 입력란에 붙여넣고 엔터를 눌러주세요. (엣지 또는 다른 브라우저 사용시 다운로드가 안될 수 있습니다)
- 다운로드한 파일의 크기·해시값이 배포처에 적힌 값과 같은지 한번 확인하면 안전합니다.

## 3단계. 설치 실행 ⚠️ 여기서 가장 많이 막힙니다
다운로드한 .exe 파일을 더블클릭하면 아래와 같은 파란 창이 뜰 수 있습니다.
> 🔵 "Windows에서 PC를 보호했습니다" 인식할 수 없는 앱이 시작되지 않도록 했습니다. (Microsoft Defender SmartScreen)
👉 공식 배포처에서 받은 파일이 맞다면 아래 절차로 여세요:
1. 창에 있는 작은 글씨 "추가 정보" 클릭
1. 아래쪽에 새로 나타나는 "실행" 버튼 클릭
> 🛡 이 경고는 새로운 앱에 대해 Windows가 항상 띄우는 일반적인 안내입니다. 단, 출처가 불분명한 파일이라면 절대 실행하지 마세요.

## 4단계. 설치 진행
설치 마법사가 시작됩니다.
- 설치 위치: 기본값 그대로 두기
- "바탕화면 바로가기 만들기": 체크 권장
- 설치가 끝나면 자동으로 Blog Pick이 실행됩니다

## 5단계. 첫 실행 시 권한 허용
처음 실행하면 다음 두 종류의 알림이 뜰 수 있습니다.
### 5-1. Windows 방화벽 알림
> "이 앱이 네트워크 통신을 허용할까요?"
- "개인 네트워크" 만 체크하고 "액세스 허용" 클릭
- 공용 Wi-Fi(카페·공항)에서는 굳이 허용하지 않아도 됩니다.
- AI(Gemini)와 통신하려면 이 권한이 꼭 필요합니다.
### 5-2. 백신 프로그램이 차단할 경우
V3, 알약, Windows Defender 등이 의심한다며 차단하는 경우가 있습니다.
- 공식 배포처에서 받은 파일이 확실하다면, 백신의 "예외 등록" 또는 "신뢰할 수 있는 항목" 으로 추가
- 잘 모르겠다면 배포처에 먼저 문의
---
## 6단계. 로그인 + API 키 등록
Blog Pick이 열리면 로그인 화면이 보입니다.
1. 가입 시 사용한 이메일/비밀번호로 로그인
1. 로그인 후 화면 우측 상단의 🔑 열쇠 아이콘 클릭 → "API 키 설정" 페이지
1. 1단계에서 저장해둔 Gemini API 키를 입력칸에 붙여넣기
1. "저장" 클릭
---
## 7단계. 네이버 블로그 계정 등록
발행할 네이버 계정을 등록합니다.
1. 네이버 ID/비밀번호 입력
1. 비밀번호는 컴퓨터에 암호화되어 저장됩니다 (외부로 전송되지 않음)
1. ⚠️ 네이버 2단계 인증을 켜둔 계정이면 발행 시 OTP 입력 창이 뜰 수 있습니다 → 휴대폰을 곁에 두세요
1. 보안을 위해 2단계 인증은 계속 켜둘 것을 권장합니다 (해제하지 마세요)
---
## 8단계. 첫 글 발행해보기
이제 준비 완료. 앱 상단의 ❓ 도움말 버튼을 클릭하면 사용 매뉴얼이 나옵니다. 거기 안내대로 한 편 발행해보세요.

## 🔄 자동 업데이트 안내
- Windows는 자동 업데이트가 활성화되어 있습니다.
- 새 버전이 나오면 앱이 자동으로 알려주고, 클릭 한 번으로 업데이트됩니다.
- 따로 새 설치 파일을 다시 받지 않아도 됩니다.

## ❓ 자주 묻는 질문 (FAQ)
Q. 설치 파일이 안 열려요 A. "추가 정보 → 실행" 절차를 다시 확인하세요. 그래도 안 되면 다운로드 출처(공식 배포처)가 맞는지 다시 확인.
Q. 백신이 바이러스라고 경고해요 A. 공식 배포처에서 받은 파일이 맞다면 백신의 예외 등록으로 처리할 수 있습니다. 출처가 불확실하면 절대 실행하지 말고 먼저 배포처에 문의하세요.
Q. 다시 설치하고 싶어요 A. 제어판 → "프로그램 추가/제거" → "Blog Pick" 삭제 후 새 설치 파일로 다시 설치하면 됩니다. 이때 등록한 계정·이미지·보관함은 그대로 유지됩니다 (앱만 다시 깔리는 것이라).
Q. Gemini API 무료 한도가 초과됐어요 A. 다음 날 자동으로 풀리거나, Google Cloud에서 결제 연결로 한도를 늘릴 수 있습니다. https://ai.google.dev/gemini-api/docs/billing 참고.
Q. 발행 중 캡차(보안 문자)가 떴어요 A. 자동화 창에 캡차가 보이면 직접 풀어주세요. 자동 우회는 약관 위반입니다.
Q. 회사 컴퓨터에서 쓰려고 합니다. 방화벽이 막아요 A. 회사 IT팀에게 다음 도메인을 허용해달라고 요청하세요:
- .googleapis.com (Gemini API)
- .naver.com, .pstatic.net (네이버 발행)
- github.com, objects.githubusercontent.com (자동 업데이트)

## 2-1. dmg 파일 받기

## 2-2. 기존 Blog Pick 종료하기
> ⚠️ 중요 — 이 단계를 빼먹으면 설치가 실패합니다.
화면 위쪽 메뉴바에 Blog Pick 글자가 보이면, 그 옆에 마우스를 갖다 대고 Blog Pick 종료 를 누릅니다. 또는 단축키 ⌘ + Q 를 누르셔도 됩니다.
> ✅ 종료됐는지 확인하는 법 화면 아래 Dock의 Blog Pick 아이콘 밑에 작은 점(●)이 없으면 종료된 상태입니다.
## 2-3. dmg 파일로 새 버전 설치하기
### 1단계 · dmg 파일 열기
다운로드된 파일(예: Blog-Pick-Mac-0.2.15.dmg)을 더블클릭합니다. → 잠시 후 창이 하나 뜨고, 그 안에 두 가지 아이콘이 보입니다.
- 왼쪽: Blog Pick 아이콘
- 오른쪽: Applications 폴더 아이콘 (또는 큰 화살표)
### 2단계 · Applications 폴더로 드래그
왼쪽의 Blog Pick 아이콘을 마우스로 꾹 눌러서 → 오른쪽의 Applications 폴더 위로 드래그 → 손 떼기.
도중에 이런 창이 뜹니다.
> 「이미 같은 이름의 항목이 있습니다. 바꾸시겠습니까?」 → 바꾸기 를 누르세요.
✅ 이게 기존 버전이 새 버전으로 교체되는 순간입니다. 복사 진행률 막대가 잠깐 떴다가 사라지면 → 설치 완료.
### 3단계 · dmg 정리 (선택사항)
설치가 끝나면 dmg 창은 닫고, 바탕화면에 「Blog Pick」 디스크 아이콘이 있으면 오른쪽 클릭 → 꺼내기 하세요. 안 꺼내도 사용에는 문제 없습니다.

Finder → 응용 프로그램(Applications) 폴더 로 가서 Blog Pick을 더블클릭합니다.
### 2단계 · 보안 경고가 뜨면 침착하게 대응하기
다음 두 가지 경고 중 하나가 뜰 수 있습니다.
> 경고 A 「Blog Pick」은(는) 확인되지 않은 개발자가 배포했기 때문에 열 수 없습니다.
> 경고 B Apple에서 검사할 수 없으며, 악성 소프트웨어가 포함되어 있는지 확인할 수 없습니다.
→ 어느 쪽이든 일단 확인 또는 완료 를 눌러 창을 닫으세요. 그리고 아래 단계를 따라 하시면 됩니다.

1. 왼쪽 메뉴에서 「개인정보 보호 및 보안」 클릭

1. 맨 아래쪽에 다음과 같은 문구가 보입니다.
> 「Blog Pick」 사용이 차단되었습니다.
1. 옆의 확인 없이 열기 버튼 누르기

→ Blog Pick이 켜집니다. 완료입니다! 🎉

# 🛠️ 3. 자주 겪는 문제와 해결법
### ❓ (Mac) 「손상되어 열 수 없습니다. 휴지통으로 이동…」 경고가 떠요.
이건 위의 경고와 다른 종류입니다. 다운로드는 멀쩡한데 macOS가 「격리(quarantine)」 표시를 잘못 붙여서 막은 경우예요.
해결 방법
1. ⌘ + Space 를 눌러 Spotlight 검색을 열고 → 터미널 입력 → 엔터
1. 검은 화면이 뜨면, 아래 한 줄을 그대로 복사해 붙여넣고 엔터
```
xattr -cr /Applications/Blog\\ Pick.app
```
1. 다시 응용 프로그램에서 Blog Pick 실행 → 정상 작동합니다.
> ✅ 무서워하지 않으셔도 돼요. 위 명령은 Mac이 잘못 붙인 「위험 표시」만 떼주는 안전한 명령입니다. 시스템을 건드리지 않습니다.
### ❓ (Windows) 「다운로드 중」에서 멈춰요.
1분 이상 진행률이 그대로면:
1. 취소 버튼 누르기
1. 인터넷 연결 확인
1. 알림이 다시 뜨면 업데이트 다시 누르기
> 그래도 안 되면 Blog Pick을 완전히 종료한 뒤 다시 실행하세요. 알림이 다시 뜹니다.
### ❓ 업데이트 알림이 안 떠요. 새 버전이 있는지 모르겠어요.
수동으로 확인하는 방법
1. 브라우저 주소창에 아래 주소를 직접 입력: https://github.com/heartblood0509-gif/App_blog2/releases/latest

- 첫 충전만 16,000원 이상으로 한 번 해주면 됩니다

이렇게 두 가지 방법이 제공될 예정이라, 블로그와 쇼츠를 함께 챙기실 수도 있고 쇼츠만 따로 만드실 수도 있어요

## 2. 영상 한 편, 직접 만들어 봅시다
만들기 전, 딱 한번만 API 키 등록 🔑
쇼츠픽은 영상을 만들 때 세 가지 AI 도구의 힘을 빌립니다 발급 방법은 각 도구마다 조금씩 다른데, 화면을 하나하나 캡처해 정리해 두었습니다. 아래 가이드를 보고 천천히 따라 하시면 됩니다.
- 글을 쓰는 제미나이 → (필독 페이지 링크)
- 목소리를 만드는 타입캐스트 → (필독 페이지 링크)
- 영상을 만드는 FAL → (필독 페이지 링크)
세 개의 키를 모두 발급받았다면, 화면 오른쪽 위의 'API 키 입력' 버튼을 눌러 각각 등록해 주세요. 등록이 끝나면 이제 영상을 만들 준비가 완료된 것입니다. 🎉🎉
쇼츠픽에 들어가면 시작할 때 두 갈래 길이 나옵니다 😳
- AI 모두 생성
- 내가 직접 제공
이 매뉴얼에서는 '내가 직접 제공'을 기준으로 안내하겠습니다. 제목과 대본을 직접 준비해 넣는 방식인데, 이 길로 만들면 결과물의 완성도가 확실히 높습니다. (왜 이 방식을 권하는지는 뒤에서 자세히 말씀드릴게요. 지금은 일단 한 편 만들어 보는 데 집중하겠습니다.)
### 1단계 · 제목과 대본 넣기
가장 먼저 영상의 제목과 대본을 넣습니다.
제목은 영상 맨 위에 두 줄로 표시됩니다. 윗줄과 아랫줄을 나눠 입력하게 되어 있는데, 이렇게 두 줄로 끊으면 짧은 쇼츠 화면에서도 메시지가 한눈에 들어옵니다. 예를 들어 윗줄에 "얼굴 빨개지는", 아랫줄에 "의외의 진짜 이유" 같은 식으로, 호기심을 끄는 한 문장을 둘로 나눠 넣으면 됩니다.
대본은 미리 준비해 둔 글을 입력창에 그대로 붙여넣으면 됩니다. (대본을 어떻게 잘 준비하는지는 뒤에서 따로 다루겠습니다. 여기서는 이미 준비된 대본이 있다고 보고 진행할게요.) 한 번에 최대 5,000자까지 넣을 수 있습니다.
대본을 붙여넣었다면, 아래 '대본 쪼개기' 버튼을 누릅니다.
### 2단계 · 장면마다 이미지·영상 채우기
이제 나뉜 장면(줄)마다 화면에 보일 이미지나 영상을 채울 차례입니다. 각 줄에는 대본 한 토막이 적혀 있고, 그 줄에 어떤 그림을 넣을지 고르면 됩니다. 오른쪽 프리뷰를 보면, 선택한 줄이 실제 영상에서 어떻게 보이는지 실시간으로 확인할 수 있습니다.
각 줄에는 채우는 방법이 몇 가지 있습니다.
- AI 이미지 생성: 그 줄의 대본 내용에 맞는 이미지를 AI가 만들어 줍니다. 줄 하나씩 생성할 수도 있고, 위쪽의 '이미지 없는 줄 일괄 AI 생성'을 누르면 아직 비어 있는 줄을 한 번에 모두 채워 줍니다.
- AI 영상 변환: 이미지가 만들어지면, 그 줄에 'AI 영상 변환' 버튼이 새로 나타납니다. 이 버튼을 누르면 정지된 이미지가 움직이는 영상으로 바뀝니다.
- 이미지 업로드 · 영상 업로드: AI에 맡기지 않고, 직접 가진 사진이나 영상을 그 줄에 올릴 수도 있습니다.
💰 AI API 사용 금액이 부담되신다면
AI 이미지 생성이나 영상 변환은 API 사용 금액이 들어갑니다. 비용이 부담되시는 분들은 한 번에 많이 충전하기보다 조금씩 충전해서 쓰시는 것을 권장드려요. 쓰는 만큼만 부담 없이 테스트해 보실 수 있습니다.
참고로 대본 쪼개기나 편집은 추가 금액이 전혀 들지 않습니다. 프로그램이 알아서 자동으로 해주기 때문에, 그 부분은 마음껏 사용하셔도 됩니다.
✅ 가장 추천하는 방법 (직접 찍은 사진·영상)
여기서 한 가지 꼭 말씀드리고 싶은 것이 있습니다. 이미지 전체를 AI로 만드는 것은 권하지 않습니다. AI가 만든 이미지는 아무래도 실사 느낌이 나지 않습니다. 그런데 이 매뉴얼을 보시는 대표님들은 모두 자신의 화장품 제품과 브랜드를 가지고 계십니다. 그 자리에서 핸드폰으로 제품을 간단히 찍기만 해도, AI가 만든 어떤 이미지보다 진짜 같고 신뢰가 가는 화면이 나옵니다. 내 제품이 직접 등장하는 영상만큼 강력한 건 없으니까요.
그래서 가장 좋은 방법은 대표님 제품을 스마트폰으로 직접 찍으시고, 부족한 부분만 중간중간 AI를 섞어 주시는 것입니다.
부담 없이 한번 테스트해 보시려면 이렇게 해보세요 👇
1. 직접 스마트폰으로 제품 영상을 찍습니다.
1. 대본은 한 문장당 3초 이하로 잡고, 한 문장씩 엔터를 눌러 줄바꿈 해줍니다.
1. 각 줄에 맞는 영상을 업로드합니다.
1. '영상 만들기'를 눌러 결과를 확인합니다.
⚠️ 직접 찍은 영상을 올릴 때 주의할 점
직접 촬영한 영상을 올릴 때 딱 하나 기억하실 것이 있습니다. 영상 길이가 그 줄의 나레이션보다 짧으면 안 됩니다.
예를 들어 그 줄의 대본을 읽는 데 5초가 걸리는데 올린 영상이 3초짜리라면, 영상이 먼저 끝나버려 나머지 2초 동안 검은 화면만 나오게 됩니다. 그러니 직접 찍은 영상은 해당 줄 나레이션보다 넉넉하게 길게 찍어 두세요.
자막 한 줄이 너무 길다 싶으면, 그 자리에서 Enter 키를 눌러 줄을 둘로 나눌 수 있고 다시 합칠수도 있습니다. 이렇게 자막과 화면을 자연스럽게 맞춰 가면 됩니다. 필요 없는 줄은 삭제할 수도 있습니다.
### 3단계 · 음성 입히기
이제 대본을 읽어줄 목소리(나레이션)를 만듭니다. 설정은 직관적입니다.
1. 음성 선택: 나레이션을 맡을 목소리를 고릅니다. 여성·남성 등 여러 목소리 중에서 영상 분위기에 맞는 것을 선택하면 됩니다. 옆의 재생 버튼(▶)을 누르면 목소리를 미리 들어볼 수 있습니다.
1. 감정/톤: 목소리의 느낌을 정합니다. 보통, 기쁨, 밝게, 슬픔, 화남, 차분하게, 속삭임 중에서 영상에 어울리는 톤을 고르면 됩니다.
1. TTS 속도: 말하는 빠르기를 조절합니다. 기본은 1.0배이고, 슬라이더를 움직여 0.5배(느리게)부터 2배(빠르게)까지 바꿀 수 있습니다
추천 설정:
1. 성우: 혜리
1. 감정/톤: 보통
1. TTS 속도: 1.1~1.2배
설정을 마쳤다면 아래 '나레이션 음성 만들기' 버튼을 누릅니다. 대본 전체에 목소리가 입혀집니다.
### 4단계 · 배경음악(BGM) 깔기
마지막으로 영상에 깔릴 배경음악을 정합니다. 방법은 두 가지입니다.
- BGM 업로드: 내가 가진 음악 파일을 직접 올립니다. (MP3, WAV, OGG 형식, 최대 20MB)
- YouTube 오디오 보관함: 유튜브가 무료로 제공하는 음악 보관함에서 곡을 고릅니다. 장르·분위기·아티스트별로 검색할 수 있습니다.
음악을 정했다면 BGM 볼륨을 조절합니다. 배경음악은 나레이션을 덮지 않을 정도로 낮게 까는 것이 좋습니다. 기본값(12%)이 적당합니다.
⚠️ 꼭 알아두실 것 (유튜브 오디오 보관함의 함정)
유튜브 오디오 보관함의 무료 음악은 오직 유튜브에서만 사용할 수 있습니다. 이 음악을 넣은 영상을 인스타그램, 틱톡 등 다른 플랫폼에 올리면 저작권 문제가 생길 수 있습니다.
그래서 권장하는 방법은 두 가지입니다.
첫째, 어디서나 자유롭게 쓸 수 있는 음악을 쓰는 것. 완전 무료 음원, 직접 만든 음악, AI로 만든 음악 등은 플랫폼을 가리지 않고 안전합니다.
둘째, 여기서 BGM을 넣지 않고 그냥 넘어가는 것. 영상을 완성한 뒤, 스마트폰으로 각 플랫폼(유튜브·인스타·틱톡 등)에 올릴 때 그 플랫폼이 제공하는 음악을 직접 입히는 방법입니다. 이렇게 하면 플랫폼별로 가장 안전하고 어울리는 음악을 고를 수 있습니다
BGM까지 정했다면, 이제 영상을 만들 차례입니다.
### 5단계 · 영상 만들고 확인하기
이제 모든 준비가 끝났습니다. 영상 제작 버튼을 누르면, 지금까지 채운 제목·대본·이미지·음성·배경음악이 하나의 영상으로 합쳐집니다.
제작이 시작되면 진행률(%)과 함께 "자막/타이틀 합성 중…" 같은 현재 작업 상태가 표시됩니다. 잠시 기다리면 "영상이 완성되었습니다!" 메시지와 함께, 완성된 영상을 그 자리에서 바로 재생해 확인할 수 있습니다.
마음에 들지 않으면, 되돌아가 고칠 수 있습니다
완성된 영상을 봤는데 수정하고 싶은 곳이 있다면, 이전 단계로 돌아가 대본을 고치거나 이미지를 바꾼 뒤 다시 만들 수 있습니다. 마음에 들 때까지 얼마든지 다듬어도 됩니다.
⚠️ 단, 이 버튼을 누르면 되돌릴 수 없습니다
화면에는 두 개의 버튼이 있습니다.
1. 영상 다운로드: 완성된 영상을 내 기기에 내려받습니다.
1. 새 영상 만들기: 처음부터 새 영상 작업을 시작합니다.
이 둘 중 하나라도 누르면, 더 이상 이전 단계로 돌아갈 수 없습니다. 프로그램은 이 버튼을 "이 영상 작업은 끝났다"는 신호로 받아들이기 때문입니다.
그러니 순서를 꼭 지켜 주세요. 영상을 충분히 확인하고, 고칠 곳이 없다는 것을 확정한 다음에 다운로드를 누르시면 됩니다. 혹시 조금이라도 고칠 곳이 있다면, 두 버튼을 누르기 전에 먼저 이전 단계로 돌아가 수정하세요.
영상을 내려받았다면, 이제 첫 번째 영상이 완성된 것입니다. 수고하셨습니다.
## 3. 왜 '직접 제공'을 권하는가
따라 해보시면서 느끼셨겠지만, 영상을 만드는 일 자체는 어렵지 않습니다. 진짜 차이를 만드는 건 어떤 대본으로 시작하느냐입니다. 그래서 앞서 미뤄둔 이야기를 지금 해보려 합니다. 왜 '내가 직접 제공'을 권하는가.
### AI는 받은 만큼만 돌려줍니다
쇼츠픽에는 두 갈래 길이 있었습니다. 'AI가 모두 생성'은 주제 키워드 몇 개만 넣으면 AI가 제목과 대본까지 알아서 써주는 길입니다. 편하긴 합니다. 하지만 여기엔 분명한 한계가 있습니다.
AI는 준 정보만큼만 결과를 냅니다. 키워드 몇 개만 던지면, AI는 그 몇 개로 짐작해 누구에게나 해당될 법한 무난한 글을 써냅니다. 대표님의 브랜드가 어떤 결을 가졌는지, 어떤 고객에게 말하고 싶은지, 어떤 이야기를 꼭 담고 싶은지, 그 어느 것도 모른 채 쓴 글입니다. 무난하지만, 남는 게 없는 글이죠.
### 좋은 대본은 '충분한 대화'에서 나옵니다
반면 ChatGPT나 제미나이 같은 AI와 충분히 대화하며 만든 대본은 다릅니다. 내 브랜드 이야기, 제품의 특징, 전하고 싶은 메시지, 참고할 자료까지 풍부하게 넣고 다듬어가면, AI는 그만큼 깊이 있고 나다운 글을 써냅니다. 같은 AI라도 무엇을 얼마나 주느냐에 따라 결과물이 완전히 달라집니다.
그래서 권하는 방식은 이렇습니다. 기획과 대본은 정보를 충분히 줄 수 있는 ChatGPT·제미나이 웹에서 제대로 만들고, 그렇게 잘 뽑아낸 제목과 대본을 쇼츠픽에 가져와 '내가 직접 제공'으로 넣는 것입니다. 좋은 재료를 가지고 와서, 영상으로 만드는 일만 쇼츠픽에 맡기는 셈이죠.
### 결국, 처음에 드린 말씀 그대로입니다
이것은 맨 처음 드린 이야기와 정확히 같습니다. 매출을 만드는 것은 뾰족한 기획이지 편집이 아닙니다. 기획(대본)은 대표님이 가장 잘할 수 있고 또 직접 하셔야 하는 일이고, 그것을 영상으로 만드는 반복 작업은 쇼츠픽에 맡기면 됩니다. '내가 직접 제공'은 바로 그 역할 분담을 그대로 실현하는 길입니다.
덧붙이면, 저는 이 흐름을 앞으로 더 매끄럽게 만들어가고 있습니다. 잘 쓴 글 한 편만 있으면 그것을 곧바로 영상용 대본으로 바꿔주는 방향으로요. 그러면 기획에서 영상까지가 한 줄기로 이어질 것입니다.
