# 📘 네이버 블로그 자동 발행 프로그램 설치 가이드

> 이 가이드는 **Mac 사용자** 기준으로 작성되었습니다.
> 비개발자도 따라할 수 있도록 작성했어요. 천천히 따라해 주세요! 😊

---

## ⚠️ 시작 전 꼭 읽어주세요

1. **네이버 자동 발행은 네이버 약관상 금지**되어 있습니다.
   - 이 프로그램은 사람이 타이핑하는 것처럼 동작하지만, **계정 정지 위험이 있습니다.**
   - **하루 최대 2개까지만 발행**하도록 제한되어 있어요.
   - **메인 계정보다는 부계정 사용**을 권장합니다.

2. **필요한 시간**: 처음 설치는 약 **1~2시간** 걸려요.
3. **필요한 것**:
   - Mac 컴퓨터
   - 네이버 블로그 계정
   - 구글 계정 (Gemini API용)
   - 신용카드 (Gemini 무료 사용량 초과 대비, 대부분 무료 내 사용 가능)

---

## 📋 전체 순서

```
1. 기본 프로그램 설치 (Homebrew, Python, Node.js)
2. 프로젝트 파일 받기
3. 프로그램 의존성 설치
4. Gemini AI 키 발급
5. 환경설정 파일 만들기
6. 크롬에 네이버 로그인
7. 실행하기
8. (선택) 자동 실행 설정
```

## 1️⃣ 기본 프로그램 설치

### 1-1. Homebrew 설치 (Mac 프로그램 설치 도구)

**터미널 앱** 을 엽니다 (Spotlight에서 "터미널" 검색).

아래 명령어를 복사해서 붙여넣고 Enter:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

비밀번호를 물어보면 **맥 로그인 비밀번호**를 입력하세요. (화면에는 안 보여요, 정상)

설치 완료까지 **5~10분** 걸립니다.

### 1-2. Python 설치

```bash
brew install python@3.12
```

### 1-3. Node.js 설치

```bash
brew install node
```

### 1-4. 설치 확인

```bash
python3 --version
node --version
npm --version
```

각 명령어가 버전 번호를 출력하면 성공입니다.

## 2️⃣ 프로젝트 파일 받기

프로젝트 폴더를 원하는 위치에 복사해 넣으세요.

예: `~/Documents/App_blog2`

터미널에서 해당 폴더로 이동:

```bash
cd ~/Documents/App_blog2
```

> 💡 폴더 경로는 본인 상황에 맞게 바꿔주세요.

---

## 3️⃣ 프로그램 의존성 설치

### 3-1. 백엔드 (파이썬) 설치

```bash
cd backend
pip3 install -r requirements.txt
python3 -m playwright install chromium
cd ..
```

### 3-2. 프론트엔드 (Node.js) 설치

```bash
cd frontend
npm install
cd ..
```

설치 시간: 약 **10~20분**

---

## 4️⃣ Gemini AI 키 발급

1. https://aistudio.google.com/apikey 접속
2. 구글 계정으로 로그인
3. **"Create API Key"** 버튼 클릭
4. API 키 복사 (예: `AIzaSy...` 로 시작하는 긴 문자열)

> ⚠️ **이 키는 비밀번호 같은 거예요.** 다른 사람에게 알려주지 마세요!

## 5️⃣ 환경설정 파일 만들기

### 5-1. 프론트엔드 환경설정

`frontend/.env` 파일을 만들고 아래 내용 입력:

```
GEMINI_API_KEY=발급받은_API_키_여기에_붙여넣기
BACKEND_URL=http://localhost:8000
```

### 5-2. 백엔드 환경설정

`backend/.env` 파일을 만들고 아래 내용 입력:

```
HOST=0.0.0.0
PORT=8000
```

## 6️⃣ 네이버 로그인 (첫 발행 시 자동 안내)

이제 **메인 Chrome에 미리 로그인할 필요 없어요.**
발행 전용 별도 Chrome 프로필을 사용하므로:

- ✅ 평소 쓰던 Chrome은 그대로 둬도 됨
- ✅ 메인 Chrome 켜져 있어도 발행 가능 (충돌 없음)
- ✅ 첫 발행 시 **새 Chrome 창**이 자동으로 뜸 → 거기서 한 번만 로그인
- ✅ 이후 발행부터는 자동 로그인됨 (세션 영구 저장)

### 첫 발행 시 흐름

1. 웹에서 발행 버튼 클릭
2. 새 Chrome 창이 자동으로 열림 (빈 프로필)
3. 네이버 로그인 페이지가 나타남
4. **"로그인 유지" 체크 후 로그인** (5분 이내)
5. 자동으로 글쓰기 페이지로 이동 → 발행 진행

> 💡 프로필 저장 위치: `~/Library/Application Support/app-blog2/chrome-profile/`
> 로그인 세션이 꼬이면 이 폴더를 삭제하면 초기화됩니다.

## 7️⃣ 실행하기

### 7-1. 백엔드 실행 (터미널 1번 창)

```bash
cd ~/Documents/App_blog2/backend
python3 main.py
```

`Uvicorn running on http://0.0.0.0:8000` 메시지가 뜨면 성공!

**이 창은 끄지 마세요.** 백엔드가 계속 실행되어야 합니다.

### 7-2. 프론트엔드 실행 (터미널 2번 창 - 새 창)

```bash
cd ~/Documents/App_blog2/frontend
npm run dev
```

### 7-3. 브라우저 접속

크롬에서 http://localhost:3000 접속

🎉 이제 사용할 수 있어요!

## 8️⃣ (선택) 맥 켜면 자동 실행

매번 터미널에서 백엔드 실행하기 귀찮다면, **맥이 켜질 때 자동으로 실행**되게 할 수 있어요.

### 8-1. 자동실행 파일 만들기

`~/Library/LaunchAgents/com.app-blog2.backend.plist` 파일을 만들고 아래 내용 입력:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.app-blog2.backend</string>
    <key>ProgramArguments</key>
    <array>
        <string>/opt/homebrew/bin/python3</string>
        <string>main.py</string>
    </array>
    <key>WorkingDirectory</key>
    <string>/Users/본인계정/Documents/App_blog2/backend</string>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>/tmp/app_blog2_backend.log</string>
    <key>StandardErrorPath</key>
    <string>/tmp/app_blog2_backend_error.log</string>
    <key>ThrottleInterval</key>
    <integer>10</integer>
</dict>
</plist>
```

> ⚠️ `/Users/본인계정/Documents/App_blog2/backend` 부분을 **본인 경로로 수정**하세요.

### 8-2. 자동실행 등록

터미널에서:

```bash
launchctl load ~/Library/LaunchAgents/com.app-blog2.backend.plist
```

이제 맥을 켤 때마다 백엔드가 **자동으로** 실행됩니다!

### 8-3. 자동실행 끄고 싶을 때

```bash
launchctl unload ~/Library/LaunchAgents/com.app-blog2.backend.plist
```

## 🆘 문제 해결

### "Python 백엔드 서버가 실행 중이 아닙니다" 에러
→ 7-1번 백엔드 실행을 다시 해보세요.

### "오늘 이미 2개를 발행했습니다" 경고
→ 계정 안전을 위한 제한입니다. 내일 다시 시도하세요.

### "로그인 시간이 초과되었습니다" 에러
→ 5분 안에 로그인을 완료하지 못했어요. 발행 버튼을 다시 누르면 새 창이 뜹니다.

### 로그인 세션이 꼬인 것 같을 때
→ 전용 프로필을 삭제하면 초기화됩니다:
```bash
rm -rf ~/Library/Application\ Support/app-blog2/chrome-profile
```
다음 발행 시 자동으로 재생성되고, 로그인만 다시 하면 됩니다.

### Gemini API 에러
→ API 키가 올바른지 확인. 무료 할당량을 초과했다면 24시간 후 재시도.
